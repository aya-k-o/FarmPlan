# 🌱 FarmPlan

畑の区画をグリッドで可視化し、野菜の配置・連作障害チェック・コンパニオンプランツの相性確認を支援するWebアプリです。

---

## 開発背景

大阪から宮崎へ移住し、農業に携わった経験からこのアプリを開発しました。
「どの区画に何を植えたか」「連作障害が出ないか」を紙やスプレッドシートで管理することに限界を感じ、視覚的に確認できるツールが必要だと感じたことがきっかけです。
完成後は自分でも使いながら家庭菜園の情報を発信していく予定です。

---

## 機能

| 機能 | 説明 |
|------|------|
| ユーザー登録・ログイン | アカウント管理・セッション管理 |
| 畑マップ（グリッド表示） | 畑の区画を1m²単位でグリッド表示。科ごとに色分け |
| 野菜配置 | 区画をクリックして野菜を登録 |
| 連作障害チェック | 同じ科の野菜を連続して植えると警告を表示 |
| コンパニオンプランツ確認 | 相性の良い・悪い組み合わせを警告表示 |
| 栽培シミュレーション | 来年の計画を立てて連作リスクを事前確認 |
| 栽培記録 | 収穫履歴・失敗記録の管理 |
| 設定 | ユーザー情報・畑サイズの変更 |

## DB構成

| テーブル | 説明 |
|----------|------|
| users | ユーザー管理 |
| fields | 畑の管理 |
| plots | 区画（1m²単位） |
| plot_seasons | 栽培履歴・連作チェックの核心 |
| vegetables | 野菜マスタ |
| companion_rules | コンパニオンプランツルール |
| harvests | 収穫記録 |

## ローカル環境構築手順

### 必要なもの
- Docker Desktop
- Git

### 手順

```bash
# 1. リポジトリをクローン
git clone https://github.com/aya-k-o/farmplan
cd farmplan

# 2. 環境変数ファイルを作成
cp .env.example .env
# .envを開いてDB接続情報を設定してください

# 3. Dockerを起動
docker-compose up -d

# 4. ブラウザで開く
# アプリ：http://localhost:8080
# phpMyAdmin：http://localhost:8888
```

## セキュリティ対応

- `password_hash()` によるパスワードのハッシュ化
- PDOプリペアドステートメントによるSQLインジェクション対策
- `htmlspecialchars()` によるXSS対策
- `session_regenerate_id()` によるセッション固定攻撃対策
- 機密情報は `.env` で管理し、Gitの管理対象外に設定
